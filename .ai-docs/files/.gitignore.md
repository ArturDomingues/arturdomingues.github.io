# `.gitignore`

## Purpose

Specifies files and directories that Git should not track. Prevents build artifacts, virtual environments, generated output, and lock files from being committed to the repository.

## Exports / Public API

Not a code file — no exports. Consumed by Git automatically.

## Internal Logic

### Ignored entries

| Pattern | What it ignores |
|---|---|
| `bin-debug/`, `bin-release/` | Binary debug/release folders (legacy ActionScript/Flash entries — not relevant to this project) |
| `[Oo]bj/`, `[Bb]in/` | Object and binary folders (legacy — not relevant) |
| `.settings/` | IDE settings folder (legacy — not relevant) |
| `*.swf`, `*.air`, `*.ipa`, `*.apk` | Flash/mobile binary files (legacy — not relevant) |
| `site/` | **Zensical/MkDocs build output** — the generated static site |
| `.cache/` | **Zensical build cache** |
| `vendor/` | Vendor dependencies directory |
| `.venv/` | **Python virtual environment** created by `uv` |
| `uv.lock` | **uv lock file** — dependency lock (gitignored, so builds are not reproducible across environments without it) |

## State & Side Effects

None — purely declarative.

## Configuration & Environment

No environment variables.

## Error Handling

Not applicable.

## How to Modify Safely

- To make the build reproducible across CI and local dev, remove `uv.lock` from `.gitignore` and commit the lock file.
- The legacy ActionScript/Flash entries (first 10 lines) are irrelevant to this project and could be removed for clarity, but they are harmless.
- Never remove `site/` from `.gitignore` — committing the build output would bloat the repository and conflict with the GitHub Pages deployment workflow.

## ⚠️ Gotchas & Known Issues

- The file contains several legacy entries from an ActionScript/Flash project template (`.swf`, `.air`, `.ipa`, `.apk`, `bin-debug/`, etc.). These are harmless but confusing — they appear to have been included from a generic `.gitignore` template and were never cleaned up.
- `uv.lock` is gitignored — see `pyproject.toml` gotchas for implications on reproducibility.
