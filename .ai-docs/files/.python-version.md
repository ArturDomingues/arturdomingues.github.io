# `.python-version`

## Purpose

Pins the Python interpreter version to **3.12** for this project. Read by `uv` (and `pyenv`) to automatically select the correct Python version when running project commands, ensuring consistency between developers and CI environments.

## Exports / Public API

Not a code file — contains a single line: `3.12`.

## Internal Logic

Single-line file:
```
3.12
```
When `uv run`, `uv sync`, or `pyenv` is invoked in this directory, they read this file and ensure Python 3.12 is used. This pairs with `requires-python = ">=3.12"` in `pyproject.toml`.

## State & Side Effects

No runtime side effects. Read by the package manager at command invocation time.

## Configuration & Environment

No environment variables. The `uv` tool uses this file automatically if present in the project root.

## Error Handling

If Python 3.12 is not installed, `uv` will fail with an error indicating the required version is not available.

## How to Modify Safely

- To upgrade Python, change the version number here **and** update `requires-python` in `pyproject.toml`.
- When upgrading, also update `.github/workflows/docs.yml` if it specifies a Python version (currently uses `3.x`, which is loose).

## ⚠️ Gotchas & Known Issues

- The CI workflow (`docs.yml`) uses `python-version: 3.x` (latest Python 3), not `3.12` specifically — so CI and local dev may use different Python minor versions. This is unlikely to cause issues for a pure static site generator, but could theoretically cause subtle dependency differences.
